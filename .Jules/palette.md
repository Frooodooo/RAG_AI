## 2024-05-18 - Double-Tap Confirmations for Destructive Actions
**Learning:** Destructive actions without confirmation (like clearing a conversation) negatively impact UX and risk accidental data loss. A common pattern in this app's components (e.g., `SessionSidebar`) is the "double-tap" confirmation (toggle confirm state on first click, execute on second, with a 3-4s timeout to reset). Re-using this pattern across all similar actions maintains interaction consistency.
**Action:** Always verify if a newly added or modified destructive action implements a double-tap confirmation pattern rather than firing immediately, and ensure the UI visually changes (e.g., using `var(--red)` and `var(--red-dim)`) during the confirmation phase to alert the user.
